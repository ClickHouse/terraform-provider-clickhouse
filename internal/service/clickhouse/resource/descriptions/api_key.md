You can use the *clickhouse_api_key* resource to create and manage ClickHouse Cloud organization API keys.

The key secret is generated by ClickHouse Cloud and returned **only once, at creation time**. It is stored in Terraform state as a sensitive value (`key_secret`). Because the API never returns the secret again, an imported `clickhouse_api_key` will have an empty `key_secret`.

To assign RBAC roles (system or custom) to the key, use the `clickhouse_role_assignment` resource with `api_key_ids`. This resource does not manage roles directly.

## Example Usage

```hcl
resource "clickhouse_api_key" "monitoring" {
  name  = "prometheus-scraper"
  state = "enabled"

  ip_access = [
    {
      source      = "10.0.0.0/8"
      description = "vpc"
    },
  ]
}
```
